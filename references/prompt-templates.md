## 1. 风格视觉指纹

用户提供截图或视频时，先用中文提取：

| 维度 | 识别内容 |
|---|---|
| 媒介 | 3D、2D、定格或混合 |
| 角色 | 年龄、体型、头身比、五官夸张程度 |
| 写实程度 | 卡通、半写实、高写实 |
| 材质 | 皮肤、眼睛、毛发、服装、金属、塑料等 |
| 灯光 | 主光、轮廓光、阴影、曝光 |
| 色彩 | 主色、辅助色、饱和度、对比度 |
| 背景 | 舞台、真实环境、抽象空间、系统界面 |
| 镜头 | 景别、视角、焦段感、景深 |
| 构图 | 主体位置、前中后景、留白 |
| 特效 | 发光、粒子、HUD、扫描、失焦、运动模糊 |
| 字幕 | 字体、颜色、描边、位置；默认后期添加 |
| 风险 | 跨景别与跨段最容易漂移的属性 |

未提供参考时使用 `pattern-library.md` 的默认视觉风格。

## 2. 主体与场景素材

第二阶段先准备可复用素材，不把人物策略写进生成Prompt。

### 全局风格母版

```text
生成竖版9:16的[媒介与风格]画面。

所有画面必须属于同一部作品和同一渲染体系。统一保持[写实程度]、[角色比例]、[皮肤/眼睛/毛发/服装材质]、[灯光]、[背景]、[色彩]、[景深]和[特效体系]。

景别、尺度和空间可以变化，但视觉媒介、角色建模、材质与光影不能改变。
```

### 人物策略卡与人物视觉母版分离

人物策略卡只供导演使用：受众映射、外部身份、内部状态、错误信念、情绪弧线、理想身份和叙事作用。不得复制到图片或视频Prompt。

人物视觉母版只保留AI能画出来的属性：年龄、脸型、肤色、眼睛、发型、体型、服装、鞋子、配饰、身份道具、姿态特征和可见情绪。多个角色分开编号。

### 主体参考图

使用人物视觉母版先生成一张高质量角色参考图。优先使用全身或四分之三身、正面清晰、无遮挡、服装完整、手部自然的画面。确认后在即梦建立主体，例如 `@主角`。产品需要跨镜头稳定复用时，另建 `@产品`。

### 场景参考图

每个主要视觉空间准备一张场景参考图。场景图可以带上 `@主角`，用来锁定人物与桌面、房间、道具或内部系统的比例关系。场景图只建立空间、比例、灯光和构图，不承载整段故事。

不要生成或上传把多个镜头拼在一张图里的分镜故事板。Seedance多帧功能使用独立关键帧、主体参考和场景参考。

### 负面约束母版

只写当前风格最可能发生的错误，例如人物变脸、服装漂移、极端特写变真人、2D图标混入3D世界、产品比例变化、错误手指、文字乱码、背景色温变化。

## 3. 图片Prompt

第一次建立主体或新场景时使用完整视觉描述。主体和场景已经上传后，后续关键帧Prompt保持简洁，只写当前人物状态、动作起止、景别和构图，不重复整套外观、风格和策略。

### 标头

以下是交付说明，不属于图片Prompt正文，不复制进生成模型：

```text
关键帧[编号]｜[时间]｜[视觉状态]
引用：[需要上传的主体、场景或独立关键帧]
用途：[为何需要独立关键帧]
```

### 完整结构

```text
使用 `@主角` 和上传的[场景参考图]，生成一张竖版9:16画面。

【人物】
写当前可见姿势、表情、视线、手部、身体重心和动作起点或终点。

【构图】
写清景别、视角、主体位置、画面占比、前中后景、焦点和字幕留白。

【必要元素】
只写本帧必须出现的道具、特效和空间状态。不要提前放入后续元素。

保持 `@主角`、场景比例和上传参考一致。极端特写仍是同一角色和同一视觉媒介。

【禁止事项】
只列当前画面最可能发生的错误，例如人物变脸、比例失调、错误肢体、额外人物、错误文字或风格漂移。
```

### 图片引用规则

- 第一张建立风格和角色母版；
- 后续人物帧引用第一张或最近的高质量同景别角色帧；
- 极端特写仍引用角色母版；
- 产品前段被动存在或中性使用时也引用 `@产品` 或产品原图，但不使用英雄构图；
- 新空间引用最近的角色帧＋空间风格帧；
- 跨10秒段的第一张原则上复用上一段末帧。

## 4. 视频Prompt

每个10秒段输出一条简洁中文Prompt。Seedance只使用实际上传的 `@主体`、场景参考图和独立关键帧，不依赖未上传的参考视频。

### 开头

```text
使用 `@主角`、`@产品`（如需要）和上传的[场景参考图/独立关键帧]，生成一条10秒、竖版9:16动画视频。

保持上传主体、场景比例和视觉风格一致。
```

不要写“复刻参考视频”“按照原视频逻辑”或其他模型无法验证的目标。

人物策略卡、Part名称、场景说服任务、心理作用和营销强度只属于导演表，不进入最终视频Prompt。

### 参考素材原则

- 已建立 `@主角` 后，不重复整段脸型、服装和材质描述；
- 已上传场景参考后，不重复整段灯光、背景和风格描述；
- 场景图最好包含主角，用来锁定人物与环境比例；
- 不上传多格拼接的分镜故事板；只上传独立关键帧；
- 极端特写仍引用 `@主角`，景别切换不是风格切换。

### 镜头编号

导演镜头表继续保留精确时间轴，用于设计节奏和选择关键帧。最终Seedance Prompt不写 `[0–3秒]` 等每镜时间戳，改用镜头编号。镜头数量比文字时间戳更能影响视频节奏。

```text
镜头01：写景别、主体和一个主要动作。

镜头02：写景别变化、发生的事件和结束画面。

镜头03：写转场后的主体、动作与构图。
```

每个镜头言简意赅，只写景别和发生了什么。一个镜头只安排一个主动作；必要时补充转场和关键音效。不要为了显得专业加入模型无法执行的摄影理论、情绪解释或运动曲线。

一个10秒生成段可以跨越两个内容Part。Part切换时间只写在导演表中；最终Prompt按镜头顺序自然体现从内容解释到解决方案关联的变化。

### Seedance完整Prompt模板

```text
使用 `@主角`、`@产品`（如需要）和上传的场景参考图，生成一条10秒、竖版9:16动画视频。保持主体身份、人物与环境比例、场景和视觉风格一致。

镜头01：[景别]，@主角[一个主要动作或状态变化]，[必要道具或特效如何出现]。

镜头02：[景别]，[发生的主要事件]，通过[硬切/遮挡/推近/跟随道具]进入下一画面。

镜头03：[景别]，@主角[结果动作或表情]；如需要建立产品关联，让@产品自然出现在画面中并与结果建立联系，不加入购买CTA。产品英雄镜头仅在需要时使用。

声音：不生成任何BGM，不生成旁白。只生成与人物动作、物体、动画特效和转场同步的音效，不加入广告式提示音。

禁止人物变脸、比例失调、额外人物、错误肢体、产品变形和关键帧之间的错误融合。
```

根据实际内容增减镜头，不机械固定为三个。Prompt应当让创作者一眼读懂每个镜头发生什么；能由主体或场景参考表达的信息，不再用文字重复。

### 高频节奏要求

高频变化阈值用于导演镜头表，不要机械复制成长段规则塞进每条Seedance Prompt。最终Prompt通过必要的镜头数量、景别变化、动作和转场体现节奏。

镜头少时，每个镜头内部可以有动作、道具进入或轻微推拉；镜头多时，减少单镜头附加动作。不要让所有元素同时剧烈运动。

### 目标语言

Prompt主体使用中文。存在角色对话、口型或画面文字时，逐字写出目标市场语言，默认英语：

```text
人物看向镜头，用自然的美国英语说：“[英文台词]”。口型与英文准确同步，不增加其他台词。
```

角色对话不改成旁白。

### 禁止事项

只列当前段最可能发生的错误，例如人物变脸、服装漂移、产品变形、额外人物、错误肢体或关键帧之间错误变形。前段允许产品简单带过或中性使用，但禁止产品英雄构图、Logo强调、销售手势、利益点展示和CTA。

## 5. 声音模块

放在视频Prompt末尾，长度应明显短于主视觉时间轴。

```text
【声音要求】

不生成背景音乐和旁白。只有脚本明确包含角色对话时，才生成目标市场语言对白。

人物动作加入自然的服装、脚步、抓握和呼吸拟音；物体进入、旋转、掠过和碰撞时加入与材质、方向和速度同步的声音。

发光、扫描、HUD、电路、粒子和能量动画加入简洁的点亮、扫描、脉冲和流动音；充能、断电、失焦、崩塌和恢复加入短促的状态音；推近、硬切、遮挡和空间穿越加入对应的转场音。

产品出现仍只使用与动作和画面一致的自然音效，不加入广告式提示音或宣传音效。只标注关键声音节点；所有音效必须有明确触发画面、开始和结束，不持续堆叠，不遮挡后期配音。
```

如果某段只有普通动作，不强行加入科技音效。声音跟随视觉语义，不为了丰富而堆砌。

## 6. 交付与检查

第二阶段依次输出：

1. 主体参考图Prompt和建议建立的 `@主体`；
2. 场景参考图Prompt；场景图可包含主角以锁定比例；
3. 独立关键帧清单与导演时间；
4. 每张关键帧的简洁图片Prompt；
5. 每个10秒段一条使用镜头编号、没有每镜时间戳的视频Prompt；
6. 跨段桥接说明。

长视频第二阶段优先逐段交付，避免一次输出几十条Prompt导致截断。用户明确要求整批输出时可以一次完成。

检查：

- 是否先准备角色主体和场景参考？
- 场景参考是否帮助锁定人物与环境比例？
- 是否避免上传多格拼接的分镜故事板？
- 后续图片Prompt是否只写可执行的人物状态、必要元素和构图？
- 是否避免把人物策略、心理作用、Part和营销强度写进Prompt？
- 极端特写是否仍是同一风格？
- 视频Prompt是否只引用实际上传的主体、场景和独立关键帧？
- 最终视频Prompt是否使用镜头01、镜头02等编号，没有每镜时间戳？
- 主体和场景已经上传后，是否避免重复长篇描述？
- 对白和画面文字是否使用目标语言？
- 是否禁止背景音乐和旁白？
- 音效是否包含动画特效与转场，而不只是真实动作？
- 声音模块是否简洁？
- 前段产品是否仅简单带过或中性使用，没有产品英雄镜头和品牌强调？
- 最后是否以人物或系统结果优先，再建立产品与解决方案的关联？
- 产品英雄镜头是否保持可选？
- 未明确要求转化时，是否没有购买CTA？
