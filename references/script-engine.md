## 1. 目标与交付

把主题、产品信息或用户原稿转成一版逻辑完整、英语自然、天然支持高频动画的最终口播稿。

保留用户的核心观点、产品信息和商业目标；允许重构钩子、顺序、因果链、转折和结尾。不要只做翻译或润色。

默认目标市场语言为英语。用户指定其他目标语言时，口播、对白和画面文字全部切换；分析、镜头表与Prompt仍使用中文。

## 2. 输入处理

### 用户提供原稿

先内部标记：

- 核心观点；
- 目标观众；
- 观众当前认知、核心欲望与主要阻力；
- 人物替身与情绪弧线；
- 表面问题；
- 原因解释；
- 旧方法；
- 认知转折；
- 产品或答案；
- 机制；
- 结果；
- 最后认知植入Part的进入点、产品关联与结果结论。

然后直接生成优化稿，不停在评价阶段等待确认。

### 用户只提供主题或产品

从零建立论证链。缺少会改变核心商业意图的信息时，最多问一个关键问题；其余信息用明确假设推进。

### 用户提供参考视频或截图

学习其论证节奏和视觉化方法，不复制具体措辞。普通参考不更新模式库，除非用户明确要求学习。

## 3. 论证链

先读取 `marketing-engine.md`，再使用这条内容优先骨架：

```text
Part A 内容钩子与代入
异常钩子
→ 观众熟悉的表面问题
→ 原因重构

Part B 解释与认知价值
→ 常见做法或失败尝试
→ 旧路径失效
→ 认知反转
→ 新方向或解决标准
→ 中性机制可视化

Part C 认知植入与解决方案
→ 产品作为前文逻辑的具体答案进入
→ 产品机制或证据
→ 新旧方案对比
→ 回到观众状态
→ 理想身份与产品认知关联
```

不机械填满节点。每条脚本至少具备：问题、原因、转折、答案、结果。

### 产品进入规则

产品必须填补前文已经建立的逻辑缺口。前段允许产品被动存在或中性使用，但默认只在最后15%–25%的Part中进行明确品牌强调、利益点总结或产品主视觉。默认不写购买CTA。

避免：

```text
你很疲惫 → 购买产品
```

优先：

```text
外部环境无法完全避开
→ 外部防御存在上限
→ 需要内部支持
→ 产品提供内部支持
```

产品进入前至少完成：观众认出自己、理解隐藏原因、看到旧方法的局限、接受新的判断标准。产品进入后只说完成结论所必需的机制、差异和结果，不重复推销。

### 对比结构

把新旧方案写成不同运动轨迹：

```text
旧方案：rushes → spikes → crashes → disappears
新方案：enters → connects → steadies → rebuilds
```

对比不仅表达观点，还要为动画提供方向、速度和状态变化。

前段对比用于理解，例如表面原因/隐藏原因、短暂刺激/稳定支持；最后认知植入Part才把通用标准与具体产品进行自然对应。

### 多种内容叙事骨架

叙事骨架是“信息按照什么顺序被观众理解”，不是画面风格。现有默认骨架属于“隐藏原因型”。根据内容选择以下一种，不要把所有结构混在一起。

#### 1. 隐藏原因型

```text
异常结果 → 熟悉问题 → 错误理解 → 隐藏原因 → 正确标准 → 结果 → 产品认知关联
```

适合健康、效率、心理和复杂问题解释。

#### 2. 人物实验型

```text
人物做出选择 → 系统发生变化 → 可见结果 → 解释原因 → 更好的做法 → 产品认知关联
```

适合触感、性能、使用方式和前后状态可以直接演示的内容。

#### 3. 误区打破型

```text
常见观点 → 反例出现 → 旧观点崩塌 → 新解释 → 正确判断标准 → 产品认知关联
```

适合已有强认知、需要教育市场的内容。

#### 4. 一天变化型

```text
早晨正常 → 输入不断累积 → 中段失控 → 找到原因 → 状态恢复 → 产品认知关联
```

适合生活方式、情绪、疲惫和日常使用场景。

#### 5. 双路径对比型

```text
同一人物分成两条路径 → 旧路径短暂改善后崩塌 → 新路径稳定变化 → 结果并列 → 产品认知关联
```

适合错误方案/正确方案、普通版本/改进版本的对比。

#### 6. 机制探秘型

```text
外部症状 → 进入内部或系统空间 → 跟随原因移动 → 关键机制改变 → 回到人物结果 → 产品认知关联
```

适合充分利用AI动画的尺度跳变、体内空间、微观系统和科技机制。

选择规则：优先选最能自然产生画面变化的一种；连续批量内容轮换骨架，避免每条都是“问题—原因—产品”。

## 4. 视觉化写作

### 每个语义单元分配一个视觉动词

优先使用可执行动词：

```text
enter, exit, rise, fall, pull, push, chase, block, dodge,
filter, surround, crack, expand, shrink, connect, unlock,
light up, shut down, travel, collide, rebuild, disappear
```

如果一句话只有抽象判断，没有人物、物体或状态能够发生变化，重写它。

### 使用空间词触发强转场

优先设计：

```text
outside, inside, underneath, above, behind, through,
deeper, within, across, back, forward
```

示例：

```text
The problem isn’t what’s happening around you.
It’s what’s missing inside you.
```

这组句子天然支持外部舞台→镜头冲向人物→进入内部空间。

### 使用对比词触发状态切换

优先设计：

```text
but, instead, not, until, then, while, because,
so, except, before, after
```

把这些词标记为镜头或状态转折候选。

### 使用三段式制造连续变化

优先采用3–4个短词并列：

```text
wired, tired, foggy
duck it, block it, filter it
slow, steady, foundational
```

每个词对应一次画面动作、图标、状态或字幕重击。

### 短句与节奏

- 一句只承载一个主要画面意图；
- 需要停顿和强调时，主动拆成独立短句；
- 避免连续三句使用同一语法结构；
- 避免连续两句都只能配静态人物；
- 每10秒至少安排一个明确的转折词或空间词；
- 前3秒必须提出异常、冲突、反常结果或身份问题。

### 视觉化检查

逐句回答：

1. 谁或什么是画面主体？
2. 主体做什么？
3. 画面状态如何改变？
4. 这句话适合人物、镜头、构图、道具、特效、尺度还是空间变化？
5. 与前一句相比，变化是否足够不同？

无法回答时重写。

## 5. 时长决策

用户指定时长时严格服从。未指定时在30–75秒内选择最短但能完成论证的时长：

| 时长 | 适用内容 |
|---|---|
| 30–39秒 | 单一观点、单一原因、一次反转 |
| 40–55秒 | 产品解释、失败尝试、简单机制 |
| 56–75秒 | 完整论证、机制、新旧方案对比和结论 |

不要为了凑时长重复同一观点。论证闭环后立即结束。

英文口播密度默认约每10秒25–38词。需要强停顿、三段式或视觉识别窗口时降低词数。

最终按10秒拆分生成段，尾段可以短于10秒。脚本的语义转折优先靠近分段边界，但不要为了整齐破坏自然语序。

## 6. 第一阶段输出

按以下顺序交付：

### A. 论证逻辑

用中文说明采用的叙事骨架，以及钩子、问题、原因、失败尝试、转折、解决标准、最后认知植入Part、产品机制、对比和结论。

### A2. 人物与营销边界

输出人物策略卡：目标受众、人物替身、外部身份、内部状态、可识别符号、情绪弧线和理想身份。它只供导演使用，不复制进图片或视频Prompt。

另行输出人物视觉母版：只保留AI能够画出来的脸、体型、服装、道具、姿态和可见情绪。内容Part与最后认知植入Part使用精确时间边界；允许在一个10秒生成段中切换Part。

### B. 内容结构问题

只写会影响说服力、视觉化或节奏的问题，避免泛泛点评。

### C. 最终版高频动画口播

使用目标市场语言，默认英语。

### D. 带时间轴口播

示例：

```text
0–3s: What happened to modern men?
3–7s: They didn’t just wake up drained.
7–10s: The inputs changed.
```

### E. 纯净配音稿

只保留最终口播，不写时间码、镜头说明、括号或中文。

### F. 中文高频镜头表

使用 `visual-engine.md` 的完整字段。

### G. 分段与关键帧计划

列出每个10秒段需要4–7张中的具体数量，并解释哪些视觉状态不能依赖连续运动生成。

第一阶段结束后等待用户确认。用户明确要求一次完成时可以继续第二阶段。

## 7. 质量检查

- 核心商业意图是否保留？
- 前3秒是否有明确钩子？
- 是否形成问题→原因→转折→答案→结果？
- 产品是否由逻辑推导出来，而不是凭空出现？
- 前75%–85%是否以内容价值为主？
- 前段产品是否仅被动存在或中性使用？
- 品牌强调、利益点总结和可选产品英雄镜头是否只出现在最后Part？
- 前段人物是否承担观众替身和理解任务，而不是推销任务？
- 默认是否结果优先、没有购买CTA，并避免广告腔与利益点堆叠？
- 是否明确选择一种内容叙事骨架，而不是机械复用隐藏原因型？
- 每个语义单元是否有视觉动词？
- 是否包含足够的空间词、对比词和三段式？
- 是否有任何连续3秒只能配静态画面？如有，重写。
- 口播词数是否匹配时长？
- 对白和画面文字是否使用目标市场语言？
- 是否同时提供时间轴稿和纯净配音稿？
