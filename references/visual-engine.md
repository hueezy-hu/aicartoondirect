## 1. 核心原则

高频变化不是频繁硬切，而是持续改变观众正在观察的状态。语义决定为什么变化，注意力节奏决定最迟何时必须变化。

每个语义单元必须对应一个可执行视觉动词。字幕只算附加刺激，不能代替人物、镜头、构图、道具、特效、尺度或空间变化。

## 2. 视频证据提取

分析上传视频时，先运行：

```bash
bash scripts/extract-video-evidence.sh '<视频路径>' '<临时输出目录>'
```

读取：

- `metadata.txt`：时长、分辨率、帧率和帧数；
- `sampled/`：按秒抽样帧；
- `scenes/`与`scene-times.txt`：场景变化候选；
- `sheets/`：全片联系表。

场景检测只能发现明显像素突变。继续人工识别没有硬切但存在姿态、图标、景别、焦点、尺度和空间变化的“感知切换”。重要节点需要在局部使用2–4fps再次抽样。

对齐口播时，不按句号机械拆分；按可视化语义单元和画面状态变化对齐。

## 3. 七条变化通道

| 通道 | 变化内容 |
|---|---|
| 人物 | 姿势、动作、表情、视线、位置、身体状态 |
| 镜头 | 景别、角度、推近、拉远、摇移、跟随、焦段感 |
| 构图 | 单人/双人、主体位置、前中后景、画面重心 |
| 道具 | 进入、退出、旋转、靠近、遮挡、聚集、破碎 |
| 特效 | 发光、粒子、轮廓、HUD、扫描、失焦、运动模糊 |
| 尺度 | 环境、人物、器官、微观、产品英雄级 |
| 空间 | 外部舞台、内部空间、系统界面、抽象空间迁移 |

连续两次变化尽量轮换通道。一个时间段只设一个主动作，其他通道作为辅助。

## 4. 三级变化节奏

### 微变化

每0.4–0.8秒至少一次：

- 眨眼、视线、手势、表情；
- 单个物体进入或退出；
- 图标点亮；
- 轻微推拉；
- 小范围焦点或光效变化。

### 中变化

每1.5–3秒至少一次：

- 明显改变两条通道；
- 新动作阶段；
- 全身景变中景；
- 单人变双人；
- 普通画面变HUD；
- 背景信息密度升级。

### 强变化

前10秒每4–7秒至少一次。20秒以上视频每6–12秒迁移视觉空间，或完成同等级注意力重置：

- 极端微距；
- 新角色或角色替换；
- 进入身体内部；
- 普通舞台变微观系统；
- 最后认知植入Part中产品可选成为英雄主体；
- 前景遮挡、失焦、闪白或速度变化；
- 构图与运动方向反转。

强变化后保留0.3–0.7秒识别窗口，不立即叠加另一个同等级事件。

## 5. 视觉空间与尺度

长视频需要“视觉章节”，不是在一个背景中无限增加图标。

可用尺度：

```text
环境级 → 人物级 → 器官级 → 微观/机制级 → 最后认知植入Part的结果级/可选产品英雄级
```

可用空间：

```text
现实/舞台 → 信息图 → HUD系统 → 内部空间 → 微观系统 → 最后认知植入Part的结果镜头
```

一个空间承担一个论证任务。切换空间时，口播应出现动作词、空间词、对比词或因果词作为触发。

### 人物、场景与对比任务

人物首先是目标观众的替身，不是主持人或销售员。按Part分配角色任务：

| Part | 人物任务 | 场景说服任务 | 可用心理作用 |
|---|---|---|---|
| A 内容钩子与代入 | 让观众认出自己 | 日常现实、压力累积、问题镜像 | 异常、好奇、自我识别、共情 |
| B 解释与认知价值 | 经历失败并理解原因 | 错误尝试、内部空间、系统解释 | 认知冲突、原因重构、理解感 |
| C 认知植入与解决方案 | 展示获得控制后的理想状态 | 产品机制、结果对比、可选英雄镜头 | 结果想象、差异确认、产品关联 |

每个主要人物建立：受众映射、外部身份、内部状态、视觉识别物、当前叙事角色、情绪弧线和理想身份。人物在前段不得主动推销产品。

每条片至少选择一条主对比轴，并把它转成动作、色彩、方向、速度或构图变化：

- 失控/掌控；
- 表面原因/隐藏原因；
- 短暂刺激/稳定支持；
- 错误路径/正确标准；
- 混乱系统/有序系统；
- 现在的自己/理想中的自己。

前两Part使用对比帮助理解，最后Part才把正确标准与具体产品明确绑定。产品可以在前两Part被动存在或中性使用。

连续3秒仅靠同尺度人物站立和字幕变化时，必须检查是否能改变动作、景别、尺度或空间。

## 6. 转场与桥接

### 语义转场

优先把强转场绑定：

- 动作词：dodge、fall、break、enter；
- 空间词：outside、inside、underneath、through；
- 对比词：but、instead、not、until；
- 因果词：because、so、then。

### 桥接元素

进入新空间时，至少考虑保留一种连续性：

- 相同主体；
- 相同道具；
- 相同主色；
- 相同运动方向；
- 相同轮廓或形状。

示例：前段使用普通生活道具作为桥接；最后Part可使用产品瓶→胶囊飞出→内部机制变化。桥接不等于提前营销。

### 转场类型选择

| 情况 | 优先转场 |
|---|---|
| 同一角色、同一空间、状态升级 | 连续运动、推拉、焦点变化 |
| 同一角色、极端景别变化 | 快速推近或硬切 |
| 媒介必须保持统一 | 明确“景别切换，不是风格切换” |
| 新空间但有桥接道具 | 跟随道具穿越 |
| 论证反转 | 硬切、遮挡、闪白、失焦再对焦 |
| 新角色/主要构图 | 新关键帧＋硬切或自然入场 |

## 7. 关键帧决策

每10秒默认5张，允许4–7张。不要平均分配。

### 优先增加关键帧

- 新角色出现；
- 进入新视觉空间；
- 极端景别；
- 最后认知植入Part中产品可选成为主要主体；
- 人物变为透明人体、器官或内部系统；
- 主要构图完全改变；
- 视觉状态不适合通过连续运动自然生成；
- 连续硬切要求两侧画面都被锁定。

### 通常交给视频Prompt

- 抬手、转头、眨眼；
- 单个物体进入；
- 轻微推近或拉远；
- 轻微颤抖；
- 图标逐个点亮；
- 道具缓慢旋转；
- 同一构图内的表情变化。

### 跨10秒段

上一段最后一张原则上复用为下一段第一张。强硬切时允许不同图片，但下一段必须继续引用全局风格母版与角色母版。

每张关键帧必须说明：对应时间、视觉状态、为什么不能只靠连续运动完成、引用哪些前序图片。

## 8. 标准拆解表

每次视频拆解或新镜头设计都使用这些字段：

| 字段 | 内容 |
|---|---|
| 时间段 | 精确到0.1秒；分析原视频时按证据精度标注 |
| 对应口播 | 目标语言原句 |
| 语义作用 | 钩子、原因、失败、反转、机制、对比、结论等 |
| 内容Part | A内容钩子与代入、B解释与认知价值、C认知植入与解决方案 |
| 人物叙事角色 | 观众替身、尝试者、理解者、结果示范者 |
| 场景说服任务 | 代入、问题镜像、失败证明、原因解释、机制、结果、行动 |
| 主对比轴 | 当前段使用的对比关系 |
| 心理作用 | 好奇、自我识别、认知反转、理解、结果想象等 |
| 营销强度 | 0内容、1自然承接、2轻营销；默认不使用3强营销 |
| 视觉空间 | 当前画面世界 |
| 景别 | 极远、全身、中景、近景、特写、极端微距 |
| 人物动作 | 主动作与辅助反应 |
| 道具变化 | 进入、退出、方向、速度、层次 |
| 特效变化 | 发光、粒子、HUD、焦点等 |
| 尺度变化 | 前后尺度 |
| 变化通道 | 七通道中实际改变的项 |
| 变化等级 | 微、中、强 |
| 视觉动词 | 一个主要可执行动词 |
| 转场方式 | 连续、硬切、遮挡、失焦、穿越等 |
| 是否需要关键帧 | 是/否 |
| 关键帧理由 | 不可连续生成的具体原因 |

以上字段属于导演表。最终图片与视频Prompt不得复制“人物叙事角色、场景说服任务、心理作用、营销强度”等抽象字段，只把它们转换成可见动作、景别和构图。

导演表继续使用精确时间轴。最终Seedance视频Prompt删除每镜时间戳，改用镜头01、镜头02等编号；一个10秒生成段跨越两个Part时，在导演表标出切换点，在最终Prompt中通过镜头顺序体现。

## 9. 质量门槛

- 每0.4–0.8秒是否至少改变一条主要通道？
- 每1.5–3秒是否出现中变化？
- 前10秒是否有强变化？
- 长视频每6–12秒是否迁移空间或完成同等级重置？
- 是否把字幕误当成主要变化？
- 是否连续使用相同通道造成机械感？
- 每段是否只有一个主动作？
- 新物体是否至少有约0.3秒识别时间？
- 强变化后是否保留识别窗口？
- 人物脸部是否长期被遮挡？
- 关键帧是否按视觉状态选择，而不是平均分布？
- 新空间是否有桥接元素？
- 极端特写是否继承相同媒介、角色和渲染体系？
- 前段人物是否是观众替身，而不是推销员？
- 每个场景是否有明确说服任务，而不是只为了换景？
- 对比是否被转成可见动作或状态，而不是只写字幕？
- 前段产品是否仅被动存在或中性使用，没有被拍成营销主体？
- 品牌强调、利益点和可选产品英雄镜头是否只在最后Part出现？
- 默认是否结果优先，没有强购买CTA？
- 最后解决方案关联是否保持自然、克制？
